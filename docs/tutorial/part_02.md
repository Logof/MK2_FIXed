# Глава 2 - Tileset



## Интерактивный
Мы упомянули интерактивные плитки. В текущей версии MTE MK2 их два: замок и передвигаемые объекты.

Если персонаж сталкивается с замком и у него есть ключ, он открывает его. Тайл "замок" исчезнет и можно будет пройти.

Передвигаемые объекты - это тайлы, которые, когда их толкает главный герой, они меняют свое положение. В игре с видом сбоку - их можно толкать только сбоку; в играх с видом сверху их можно толкать в любом направлении.

## 
Первое, с чем нужно определиться - будет использоваться набор из 16 или 48 тайлов. При использовании 16 тайлов, все они будут представлены 4-битным числом. Это означает, что в одном байте, будет храниться два тайла, поэтому карты будут занимать ровно половину памяти.

При этом, как будет показано позже в этой главе, использование 16 тайлов позволит активировать автоматический эффект тени, из-за чего будет казаться, что используется больше 16 тайлов.

Можно сделать короткие многоуровневые игры, в которых будут переключаться набор тайлов для каждого уровня. Это даст достаточно разнообразия и позволит использовать карты половинного размера.

Для создания tileset можно использовать любой графический редактор. В графическом редакторе нужно создать файл размером 256 × 48 пикселей. Для удобства лучше настроить отображение сетки с шагом 16 × 16 пикселей. Это поможет создавать графику в соответствии с ограничениями Spectrum или узнать, где начинается и заканчивается каждый тайл.

![img.png](https://raw.githubusercontent.com/mojontwins/MK1/master/docs/wiki-img/02_empty_ts.png)

(???)Что касается палитры Spectrum, как и всего этого, значения по умолчанию, поддерживаемые преобразователями, включенными в цепочку инструментов, довольно произвольны. Чтобы все прошло хорошо, используйте значения R, G, B 200, если вы хотите представлять цвета без ЯРКОСТИ и 255, если вы хотите представлять цвета с ЯРКОСТЬЮ. Mappy не любит глубокий пурпурный (255, 0, 255), поэтому для этого цвета вы можете использовать, например, (254, 0, 255).

## Изготовление tileset из 16 тайлов
Для экономии памяти используются наборы тайлов с 16 фрагментами, вроде этого:

![img.png](https://raw.githubusercontent.com/mojontwins/MK1/master/docs/wiki-img/02_tileset_16.png)

Набор тайлов разделен на две части: первая, состоящая из первых шестнадцати плиток - это то, что будет использовать для создания карт. 
Вторая часть, состоящая из четырех тайлов - это специальный раздел, который будет использоваться для отображения особых игровых вещей.

## Карта
По соглашению, тайл 0, то есть первый тайл, будет основным тайлом фона. Это не обязательное требование, но это упрощает жизнь: при создании пустой карты все тайлы уже будут установлены на 0.

Тайлы с 1 по 13 могут быть любыми: фоновые плитки, препятствия, платформы, враги ...

Тайл 14 (предпоследний): передвигаемый тайл. Он должен быть 14-м и никаким другим.

Тайл 15 (последний), используется как замок. Он должен быть 15-е, и никаким другим.

Если не требуется использовать кнопки или ключи/замки, то можно свободно использовать тайлы 14 и 15.

## Специальный раздел
Он состоит из четырех плиток (слева направо):

Плитка 16 - это пополнение жизни. Он расположен на карте и, если игрок "возьмет" его, то пополнится немного жизни.

Плитка 17 представляет собой предметы коллекционирования, которые игрок собирает во время игры.

Плитка 18 используется как ключ. Если используются ключи и замки, то ключ изображается в этом тайле.

Плитка 19 - альтернативный фон. Чтобы дать экранам немного уникальности, случайным образом этот тайл будет время от времени использоваться вместо тайла 0. 
Например, если ваш тайл 0 - это небо, можно поставить звезду на 19 тайл. Или, Альтернативный фон работает только на наборах из 16 тайлов. (???)Начиная с версии 5.3, можно деактивировать или выбрать другую плитку.

Обычно требуется нарисовать 20 тайлов: 16 для создания карты и 4 для изображения объектов и уменьшения монотонности фона.

Во-первых, здесь нет явного различия между разделом карты и специальным разделом - это все вместе. ???

## У нас уже есть нарисованный тайлсет. Что теперь?
MTE MK2 использует набор из 256 тайлов для «фиксированных» частей (фон, тексты, маркеры). «Узор» или символ - это не что иное, как изображение размером 8x8 пикселей. 
Из 256 «тайлов» первые 64 будут содержать буквы, цифры и символы, а остальные 192 - будут содержать различные части, используемые для построения нарисованного вами набора тайлов.

Чтобы все биты были такими, какими они нужны, MTE_MK2 имеет конвертер под названием ts2bin, расположенный, как и все утилиты, в каталоге src/linux/utils. 
Для работы ts2bin требуется ДВА файла: один с 64-мя символами шрифта, а другой с tileset.

Если вы не хотите ничего изменять, то первое, что вам нужно сделать, это взять свой набор тайлов (tileset), называть его work.png и сохранить его в /gfx. Затем нужно сделать файл с исходником.

## Шрифты
Файл со шрифтом должен называться font.png и находиться в каталоге gfx. 
Если вы не хотите редактировать build.sh, то файл со шрифтом должен содержать 64 тайла, которые представляют изображения символы с 32 по 95 кодировки ASCII и хранится в виде изображения 256x16, подобного следующему:

![font_example](https://raw.githubusercontent.com/mojontwins/MK1/master/docs/wiki-img/02_base_font.png)

Самый простой способ - переименовать файл base_source.png, включенный в пакет, в качестве шаблона и рисовать на нем или адаптировать существующий шрифт.

## Конвертация и импорт
Утилита ts2bin, включенная в /utils, берет файл со шрифтом и набор тайлов и генерирует двоичный файл со всеми шаблонами в формате, ожидаемом MTE MK2.
Вызов конвертера включен в make.sh.

ts2bin предназначен для вызова из командной строки (или из файла сценария, такого как make.sh). 
Если вы запустите его без параметров из командной строки, ts2bin сообщит вам, какие параметры он ожидает:

```
$ src/utils/ts2bin
ts2bin v0.4 20200119 ~ Usage:

    $ ts2bin font.png/nofont work.png ts.bin [forcezero]

    where:
       * font.png is a 256x16 file with 64 chars ascii 32-95
         (use 'nofont' if you don't want to include a font & gen. 192 tiles)
       * work.png is a 256x48 file with your 16x16 tiles
         (use 'notiles' if you don't want to include a tileset & gen. 64 tiles)
         (use 'blank' if you want to generate a 100% blank placeholder tileset)
       * ts.bin is the output, 2304 bytes bin file.
       * forcezero: adds 0 as 2nd colour when there's only one per 8x8 cell
```
Первый параметр - это имя файла со шрифом или nofont, если вы хотите только преобразовать набор тайлов.

Второй параметр - это имя файла с набором тайлов или notiles, если вы хотите преобразовать только шрифт.

Третий параметр - это имя файла, в который получится на выходе.

Четвертый параметр является необязательным и используется для указания цвета чернил 0-7, который вы хотите использовать, если будет обнаружен какой-либо «узор», представляющий собой сплошной квадрат в качестве второго цвета. Если вы знаете о Спектруме, вы поймете это.

Если открыть make.sh, можно увидеть, что параметры, используемые в вызове для получения шаблонов, соответствуют указанным выше (шрифт называется font.png, а файл с tileset - work.png, и они находятся в gfx). Выходной файл - tileset.bin, а 7 используется как цвет по умолчанию:
```
../utils/ts2bin ../gfx/font.png ../gfx/work.png ../bin/ts.bin forcezero
```
